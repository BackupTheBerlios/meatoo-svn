Frequently Asked Questions about Meatoo

  1. Q: Package "foo" on Gentoo isn't the same software as package "foo" on Freshmeat, why isn't it mapped to the correct name?

    A: Meatoo maps Gentoo portage package names against Freshmeat package names automatically. If you find an incorrect match, click the edit icon to re-map the proper category/packagename to the Freshmeat name.

  2. Q: Can I get package release info via mail?

    A: Not at the moment, we have no smtp server here. Got a server we can use? There is an RSS feed available for daily updates for all pacakges:

    http://gentooexperimental.org/static/meatoo.xml

    Or daily by herd or maintainer email address:

    http://gentooexperimental.org/meatoo/rss/games
    http://gentooexperimental.org/meatoo/rss/whoever@gentoo.org

    Note: Freshmeat only updates their RDF database every 12 hours (12am and 12pm UTC), so there is no need to set your client to read them at a lower interval.

    There is also a command line tool being worked on but is currently broken.

  3. Q: I love to file bug reports. Can I start filling up bugs.gentoo.org with version bump bugs?

    A: Please don't. A Gentoo developer may already be testing a package just released. If its been a reasonable amount of time, feel free to open a bug report requesting a new version.

  4. Q: Whats all this 'trove' stuff?

    A: The troves are Freshmeats categorie. You can browse them to see which packages are not in portage.


Miscellaneous:

Join #meatoo on Freenode and talk to pythonhead and rl03 if you have ideas or meatoo problems.

The meatoo database is populated with a Python script via cron once a day using celementtree to parse the ~100 meg Freshmeat RDF file and sqlobject to fill up a mysql database. The Meatoo website uses CherryPy and Cheetah.
